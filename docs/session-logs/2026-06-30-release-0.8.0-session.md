# 2026-06-30 - Release 0.8.0 Session

## Objective

Implement the first simulated EMC test execution workflow with blocking
metrology preflight, persisted refused/completed attempts, audit, outbox, API,
Python client, Qt operator flow, tests, and release documentation.

## Changes

- Added project migration `0003_simulated_test_executions.sql`.
- Added Rust `test_execution_*` modules for DTOs, repository persistence, and
  workflow service orchestration.
- Refactored project sync operation fingerprinting so the outbox can represent
  `simulated_test_execution` entities instead of forcing every project-domain
  operation to look like a `project`.
- Exposed `POST /api/v1/test-executions/simulated-emc`,
  `GET /api/v1/test-executions/{attempt_id}`, and
  `GET /api/v1/projects/{code}/test-executions`.
- Added structured refusal causes with business dimensions.
- Added deterministic simulated conducted-emission result persistence.
- Added Python local-agent client and GUI action support.
- Added Qt form/table support for the single operator workflow.
- Added real HTTP E2E coverage for refusal, correction, completion, replay,
  conflict, restart persistence, audit, and outbox.
- Bumped the software version to `0.8.0`.

## Boundaries

- No real drivers, acquisition, FFT, report, file upload, planning reservation,
  object storage, PostgreSQL, Parquet, or HDF5 was added.
- No broad Qt rewrite was attempted.
- The workflow remains one intentionally narrow simulated EMC test type.
