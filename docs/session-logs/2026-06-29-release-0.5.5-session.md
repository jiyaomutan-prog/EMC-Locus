# 2026-06-29 - Release 0.5.5 Session

## Objective

Move the Qt/Python project read path for the migrated project vertical slice to
the local agent whenever `agent_url` is configured.

## Changes

- Added `LocalAgentClient` methods for project list/detail, contract-review
  status, project audit events, and sync outbox reads.
- Added agent-backed project bootstrap generation for Qt/browser payloads.
- Updated Qt console data loading so project data comes from the agent in
  `agent_url` mode even when legacy SQLite paths are also provided for other
  domains.
- Updated project create/review/transition refreshes so `bootstrap_output` can
  be regenerated from agent reads without requiring `projects_db`.
- Added Python tests for agent project reads, agent-backed GUI refreshes, agent
  write routing, and Qt console project view refresh from agent data.
- Bumped the software version to `0.5.5`.

## Boundaries

- Service planning remains legacy SQLite until it has an agent route.
- Metrology, test definitions, measurement data, update catalog, instruments,
  acquisition, reporting, central sync, PostgreSQL, and object storage remain
  outside this tranche.
