# 2026-07-01 - Release 0.9.2 Session

Objective: fix the Windows launcher failure caused by paths containing spaces,
without adding Template Studio or new runtime functionality.

Completed work:

- Replaced fragile launcher argument handling with relative process arguments
  and repository `WorkingDirectory`.
- Added shared launcher utilities for command discovery, HTTP readiness,
  storage-root validation, PID state, safe process matching, and stop behavior.
- Hardened `start-proto.ps1` with `py` checks, `index.html` validation, port
  checks, HTTP 200 readiness, stderr reporting, and delayed browser opening.
- Hardened `start-agent-qt.ps1` with `cargo`/`py` checks, direct agent
  executable launch, storage init verification, health wait, storage-root
  comparison, incompatible-agent refusal, and `-NoQt` smoke mode.
- Added explicit `start-qt-demo.ps1 -Mode Static|Agent|Auto` behavior.
- Added stop scripts for prototype, agent, and all launcher-owned processes.
- Added `scripts/smoke-launchers.ps1`.
- Updated README, GUI-shell README, local-agent docs, changelog, revision
  control, release notes, and version files.
