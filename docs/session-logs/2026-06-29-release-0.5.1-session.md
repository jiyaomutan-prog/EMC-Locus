# 2026-06-29 - Release 0.5.1 Session

## Objective

Harden idempotent replay for the existing agent-backed project vertical slice.

## Changes

- Reused the existing `sync_operations.payload_checksum` field as the canonical
  operation fingerprint for this slice.
- Fingerprint input now includes project domain/entity identity, operation kind,
  base revision, actor/device/correlation metadata, and canonical payload.
- Replays with identical operation id and canonical fingerprint are accepted.
- Replays with the same operation id and different payload or metadata are
  rejected with `operation_replay_mismatch` and structured fingerprint details.
- Added Rust and Python tests for replay success and conflict mapping.

## Boundaries

- No schema migration in this slice.
- No Qt project-read migration yet.
- Manual JSON rendering remains for the next 0.5.2 tranche.
