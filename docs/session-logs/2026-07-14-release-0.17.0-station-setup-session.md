# 2026-07-14 - Release 0.17.0 Station Setup Session

## Goal

Replace the ambiguous measurement-engineering presentation with a useful
operator workflow that defines real equipment inputs/outputs and retains the
correct signal correction for the physical material used.

## Delivered

- Confirmed and documented the domain split between time-sample conversion,
  frequency-response compensation and sensor power/conditioning.
- Added a revisioned Rust station-setup aggregate and separate SQLite domain.
- Added Local Agent services, repositories, routes, audit, operation replay and
  outbox persistence.
- Linked real metrology assets to pinned equipment-model ports and
  serial-specific characterization checksums.
- Added structured readiness with distinct missing-evidence and
  nonconformity dimensions.
- Added Python client and Qt shaping helpers.
- Replaced agent-backed Qt dashboard/forms with the focused
  `Préparation du poste` workflow.
- Added a real RF cable example with Fmin/Fmax ports and measured loss response
  selected for the serialized cable.

## Review Findings Corrected

- The Python registration client still forced a legacy metrology category on
  passive model-backed assets; category is now optional on that path.
- Missing evidence and nonconformity were initially collapsed into calibration
  readiness; they now remain explainable dimensions.
- The first Qt material table overflowed at 1280 x 720; identity, serial number
  and aptitude now share one responsive cell with tooltips.
- The correction selector did not follow the characterization already saved in
  the setup; it now restores the pinned selection.
- The station identity foreign key now uses a deferred revision reference so
  identity and first revision can be created atomically.

## Limits

- No real acquisition, correction application, FFT, execution-package binding,
  reporting, RBAC or central synchronization was added.
- Readiness validates every selected serial-specific correction, but the model
  does not yet declare that a given signal path requires one automatically.
- The generic internal route/type names for historical correction aggregates
  remain stable; the operator vocabulary is intentionally different.
- Automated tests and screenshots do not establish laboratory validation or EN
  ISO/IEC 17025 conformity.
