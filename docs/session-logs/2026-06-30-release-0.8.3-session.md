# 2026-06-30 - Release 0.8.3 Session

## Goal

Move the documented template backbone into one small executable backend slice:
agent-owned draft test-template creation.

## Scope

- Implement one controlled test-template draft workflow.
- Use the Rust local agent as the write boundary.
- Persist to `test_definitions.sqlite`.
- Emit audit and outbox evidence.
- Add Python client coverage.

Out of scope:

- method approval;
- template lifecycle transitions;
- project/campaign instantiation;
- TEST CONSOLE execution package generation;
- instrument runtime, acquisition, FFT, reporting, or final sync.

## Changes

- Added test-definition migration `0003_test_templates.sql`.
- Added Rust `test_template_dto`, `test_template_repository`, and
  `test_template_service` modules.
- Added local API routes under `/api/v1/test-templates`.
- Added Python `LocalAgentClient` methods for test-template create/list/get/audit.
- Updated agent storage initialization to include `test_definitions.sqlite`.
- Added `docs/test-template-api.md`.
- Bumped the software version to `0.8.3`.
