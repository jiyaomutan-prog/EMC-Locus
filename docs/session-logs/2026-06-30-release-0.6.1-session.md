# 2026-06-30 - Release 0.6.1 Session

## Objective

Introduce the first metrology hardening tranche for the 0.7.0 vertical slice:
separate instrument serviceability from legacy availability and planning
reservation.

## Changes

- Added metrology migration `0004_instrument_serviceability.sql`.
- Added Rust `InstrumentServiceability` and updated readiness blocking to use
  serviceability instead of `availability`.
- Preserved legacy `availability`, `reserved`, and `out_of_service` data through
  a non-destructive migration.
- Added Python repository and action support for serviceability updates.
- Updated Qt view models, Qt action execution, static GUI rows, and bootstrap
  data to show `Service` and `Planning` separately.
- Bumped the software version to `0.6.1` and added release documentation.

## Boundaries

- No Rust metrology agent routes were added in this tranche.
- Calibration decisions, computed calibration status, readiness DTOs,
  metrology audit/outbox, and HTTP E2E tests remain in later 0.6.x tranches
  toward `0.7.0`.
