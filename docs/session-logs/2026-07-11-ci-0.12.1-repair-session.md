## Root Cause

The CI runner checked out LAB CONSOLE HTML as CRLF on Windows. Vite regenerated
`apps/lab-console/dist/index.html` with LF output during `npm run build`, so
`git diff --exit-code -- apps/lab-console/dist` detected a modified generated
bundle even though the TypeScript, tests and build succeeded.

This was a release-pipeline determinism issue, not a LAB CONSOLE runtime issue.

## Fix Plan

- Add `.gitattributes` rules to keep LAB CONSOLE source and generated text
  artifacts on LF line endings.
- Keep the dist verification check enabled.
- Add a local `scripts/validate-ci.ps1` that follows the CI command sequence.
- Standardize active release validation on npm and remove the active pnpm
  rebuild fallback.
- Add CI tool-version printing and failure artifact upload.
- Bump synchronized versions and release docs to `0.12.1`.

## Post-Push CI Follow-Up

The first pushed 0.12.1 run reached the Rust dependency download phase and
failed in `Run Clippy` before product validation could continue:

- Workflow run id: `29161927260`
- Job id: `86568354475`
- Step: `Run Clippy`
- Command: `cargo clippy --workspace --all-targets -- -D warnings`
- First relevant error: `failed to download from
  https://static.crates.io/crates/zerocopy/0.8.52/download`
- HTTP status: `503 Service Unavailable`

This was a transient crates.io/static CDN availability failure rather than a
code or model failure. The GitHub integration available in this session could
read logs but could not rerun jobs (`403 Resource not accessible by
integration`), so the workflow was hardened with:

```yaml
CARGO_NET_RETRY: "10"
CARGO_HTTP_TIMEOUT: "120"
```

The checks themselves were not removed or weakened.
