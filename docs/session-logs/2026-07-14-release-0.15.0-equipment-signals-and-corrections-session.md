# 2026-07-14 - Release 0.15.0 Equipment Signals And Corrections Session

## Objective

Audit and refound the confusing measurement-engineering surface around the
actual purpose: equipment inputs, outputs, sensor conditioning, and controlled
correction of time-domain samples or frequency-domain spectra.

## Work Completed

- Replaced the operator vocabulary with signal-centered French terms.
- Added typed time-domain representation, gain/offset explanation, and
  overload/clipping limits to sample conversions.
- Added typed amplitude/phase components and explicit correction operations to
  frequency responses.
- Added revision-pinned signal paths to equipment definitions and validation of
  path topology, port direction, communication-port exclusion, and correction
  references.
- Added agent-side resolution of correction identity, controlled revision, and
  checksum on all equipment-model write and lifecycle paths.
- Added readable port cards, advanced electrical disclosure, logical result
  outputs, RF Fmin/Fmax defaults, and a usable signal-path editor to LAB
  CONSOLE.
- Localized sensor power/conditioning and chain labels, and removed remaining
  operator dependence on internal `scaling` and generic engineering-curve
  terms.
- Hardened the local HTTP loop so an invalid non-UTF-8 request body receives a
  structured `400` response without stopping the agent.
- Added core, HTTP integration, React, and Playwright regressions.

## Modeling Notes

- Additive definition fields retain their existing schema-family identifiers
  and serde defaults, so stored 0.14.0 revisions remain readable. A subsequent
  edit canonicalizes the explicit representation and correction semantics in
  the new revision checksum.
- A frequency response needs a frequency axis and exactly one amplitude
  component; phase is optional and angular.
- A model path can only reference an approved or superseded controlled
  definition. The exact revision checksum prevents silent retargeting.
- Physical asset identity stays in metrology. The asset already pins the
  approved model revision, allowing its declared paths to be traced without
  duplicating model definitions in metrology.
