# 2026-06-30 - Release 0.8.4 Session

## Goal

Make agent-owned test templates more usable by adding the first controlled
lifecycle transitions after draft creation.

## Scope

- Add submit-for-review and approve transitions.
- Keep the Rust local agent as the write boundary.
- Reuse `test_definitions.sqlite` 0.8.3 schema.
- Emit audit and outbox evidence for each transition.
- Add Python client coverage.

Out of scope:

- configurable second approval;
- role/competence authorization;
- suspension, retirement, or supersession;
- campaign/test-instance instantiation;
- TEST CONSOLE execution package generation;
- instrument runtime, acquisition, FFT, reporting, or final sync.

## Changes

- Added `TransitionTestTemplateStatusInput`.
- Added `transition_test_template_status`.
- Added repository status-update support for `test_templates`.
- Added local API routes under
  `/api/v1/test-templates/{template_id}/transitions/...`.
- Added Python `LocalAgentClient` methods for submit and approve transitions.
- Updated API, storage, local-agent, changelog, and README documentation.
- Bumped the software version to `0.8.4`.
