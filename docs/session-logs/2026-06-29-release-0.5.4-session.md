# 2026-06-29 - Release 0.5.4 Session

## Objective

Make the project/sync multi-SQLite atomicity policy explicit, executable, and
visible through storage status before continuing toward the Qt read migration.

## Changes

- Added a shared Rust SQLite policy module for project-slice journal-mode
  checks.
- Made `storage init` set `projects.sqlite` and `sync.sqlite` to rollback
  `journal_mode=DELETE`.
- Extended storage status reports with `journal_mode` and
  `atomicity_compatible`.
- Made project repository opening refuse incompatible journal modes before
  attached-database project writes can run.
- Added Rust coverage for incompatible WAL status/verify behavior and project
  command refusal.
- Documented the decision in ADR 0003.
- Bumped the software version to `0.5.4`.

## Boundaries

- The outbox remains in `sync.sqlite`; no schema move was made in this tranche.
- No Qt migration, CI workflow, metrology, planning, instrument, acquisition,
  reporting, central sync, PostgreSQL, or object-storage work was added.
