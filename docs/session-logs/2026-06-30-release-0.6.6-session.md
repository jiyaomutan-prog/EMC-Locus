# 2026-06-30 - Release 0.6.6 Session

## Objective

Migrate the temporary Qt/Python metrology surface to the local Rust agent while
keeping Rust as the source of truth for metrology rules.

## Changes

- Added Python local-agent client methods for metrology list/detail,
  calibrations, computed status, readiness, and metrology audit events.
- Routed Qt/Python metrology bootstrap through the agent when `agent_url` is
  configured.
- Added a simple Qt readiness table populated from the agent readiness route.
- Routed instrument registration, calibration-event recording, and
  serviceability changes through the agent without opening `metrology.sqlite`
  in agent mode.
- Forwarded certificate document-manifest metadata to agent-backed calibration
  events.
- Kept the standalone document form legacy-only until a dedicated document API
  route exists.
- Bumped the software version to `0.6.6`.

## Boundaries

- No Qt rewrite, no web metrology app, and no object-storage upload were added.
- Standalone instrument documents remain legacy direct SQLite.
- Historical migration and full HTTP restart E2E coverage remain in the next
  tranche.
