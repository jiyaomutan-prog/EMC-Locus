# 2026-06-30 - Release 0.8.2 Session

## Goal

Freeze the GUI and template backbone before adding another runtime slice.

## Scope

- No Rust runtime refactor.
- No SQLite migration.
- No instrument driver, acquisition, FFT, reporting, object upload, or final
  synchronization work.
- Static GUI shell changes only for information architecture.

## Decisions

- LAB CONSOLE is the web-oriented laboratory management console.
- TEST CONSOLE is the Qt local/offline execution console.
- Metrology is a controlled LAB domain and a TEST readiness dependency, not a
  third GUI product.
- The static web shell is not the final LAB CONSOLE architecture and must not
  host TEST CONSOLE runtime behavior.
- Templates, methods, documents, people, roles, competences, approvals,
  execution packages, and result-state lineage must be stabilized before the
  next major runtime vertical.

## Changes

- Updated the GUI architecture, LAB information architecture, TEST workspace,
  template/execution backbone, and people/roles/competence documentation.
- Accepted ADR 0002 as the active LAB/TEST console split and template backbone
  decision.
- Updated ADR 0003 so the document registry remains valid while the old
  three-GUI-product wording is superseded.
- Refactored `apps/gui-shell` around hierarchical LAB navigation, console
  boundary cards, LAB-to-TEST handoff notes, object relationships, and explicit
  guardrails against fake runtime behavior.
- Bumped the software version to `0.8.2`.
