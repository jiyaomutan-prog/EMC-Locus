# 2026-07-11 - Release 0.12.0 Session

## Objective

Deliver `0.12.0 - Equipment Physics Classification Productization` as one
coherent release from the `0.11.0` equipment catalog baseline and the
physics-classification work already present on `main`.

## Work Completed

- Preserved the existing equipment catalog, driver profile, service-planning
  hardening and CAN bus vocabulary work.
- Added equipment migration `0002_physical_classification.sql` with registries,
  classification presets, preset ports, indexed summaries, and seed data for
  representative CEM/electronics measurement-chain equipment.
- Added local-agent repository, service, DTO and route support for
  classification registries, preset listing/detail, preset-based draft model
  creation, and indexed model filters.
- Updated transaction paths so model creation, draft replacement, revision
  derivation, submit and approve refresh classification summaries from the
  canonical Rust definition.
- Extended the Python client for registries, presets, preset creation and
  filtered equipment model listing.
- Expanded LAB CONSOLE Equipment workspace with backend filters, preset
  creation, preset preview, Classification editing and Port Topology editing.
- Hardened core validation for ADC/DAC/CAN bus ambiguity, CAN bus protocol
  topology, role/class coherence, RF impedance, through devices, sensors,
  measurement instruments, acquisition devices and software-only models.
- Updated release/version docs, storage docs, equipment API docs, domain docs,
  roadmap, changelog and LAB CONSOLE README.

## Validation Plan

Final validation for the committed release must include:

- `cargo fmt --check`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace`
- `py -m compileall python\emc_locus`
- `py -m py_compile apps\qt-console\main.py`
- `PYTHONPATH=python py -m unittest discover -s python\tests`
- SQLite migration validation helper
- LAB CONSOLE `tsc --noEmit`, `eslint .`, `vitest run`, and production build
- Playwright E2E where the local browser/runtime is available
- launcher/equipment smoke checks
- `git diff --check`
- `git diff --cached --check`

## Limits

This release productizes classification and preset authoring. It does not
implement the measurement-chain builder, physical asset fleet binding, real
instrument sessions, DAQ acquisition, sensor scaling, antenna-factor or
cable-loss curves, reports, RBAC, or central synchronization.

The historical `apps\gui-shell\app.js` syntax check is not applicable to this
release because the static GUI shell was removed from the product path when LAB
CONSOLE became the served browser surface.
