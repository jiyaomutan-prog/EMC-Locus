# 2026-06-29 - Release 0.5.0 Session

## Objective

Finalize the first agent-backed project vertical slice as version `0.5.0`.

## Release Contents

- Agent-owned project storage initialization and verification.
- Transactional project writes with audit and pending sync outbox records.
- Versioned loopback API under `/api/v1`.
- Thin Python local-agent client.
- Qt project forms able to call the agent when `--agent-url` is configured,
  with visible local-agent storage status and worker-backed submission.
- Real HTTP E2E test covering operation replay, restart, and persistence.
- API coverage for `storage/status`, invalid JSON, `non_accredited`, and
  `investigation` project gates.

## Boundaries

- No PostgreSQL backend.
- No central sync server.
- No object storage.
- No instrument/acquisition/signal/reporting migration in this release.
- Non-project Qt workflows remain legacy direct SQLite.
