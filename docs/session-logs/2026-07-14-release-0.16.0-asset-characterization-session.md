# 2026-07-14 - Release 0.16.0 Asset Characterization Session

## Goal

Turn `Matériels réels` into a useful metrologist workflow and let one physical
asset retain the correction measured for its own serial number.

## Delivered

- Defined the metrologist workflow before implementation in
  `docs/domain/asset-characterization-workflow.md`.
- Added a typed canonical Rust aggregate reusing the distinct time-sample and
  frequency-response contracts introduced in 0.15.0.
- Added immutable metrology persistence, operation idempotence, audit, outbox,
  HTTP/CLI routes and content-addressed proof storage.
- Added Python and LAB CONSOLE client contracts.
- Replaced the registration-only physical-assets screen with a selected asset
  dossier, status summary, history, guided entry and readable result detail.
- Added unit, real HTTP restart, file-storage, wrong-asset audit, Python client,
  React and Playwright coverage.

## Review Findings Corrected

- Storage initialization still expected metrology migration 8; it now expects
  migration 9.
- The characterization audit route originally ignored the asset id in its URL;
  it now verifies ownership before returning the journal.
- The first 1280 x 720 layout stacked status blocks unnecessarily and a sticky
  action bar obscured form content; the final layout keeps desktop status in
  one row and uses a natural end-of-form action bar.
- E2E setup initially crossed the equipment and metrology category taxonomies;
  it now uses a complete typed equipment-model reference for the RF cable.

## Limits

- The runtime does not yet select or apply the recorded correction.
- No real instrument acquisition, DAQ, FFT, station wiring, reporting, RBAC or
  central synchronization was added.
- Calibration certificate upload outside this characterization workflow remains
  incomplete.
- Automated evidence does not demonstrate laboratory usability or EN ISO/IEC
  17025 compliance.
