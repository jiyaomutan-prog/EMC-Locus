# 2026-07-01 - Release 0.9.0 Session

## Objective

Replace the superficial 0.8.x test-template model with a typed, revisioned
business aggregate suitable for a future LAB CONSOLE editor.

## Work Completed

- Added `emc-locus-core::test_definitions` with typed definition structures,
  validation, canonical JSON serialization, and SHA-256 checksums.
- Added SQLite migration `0004_template_revision_aggregate.sql`.
- Replaced the agent test-template repository, service, DTOs, and HTTP routes.
- Added optimistic draft replacement with `expected_definition_checksum`.
- Added immutable `under_review` and `approved` behavior.
- Added derivation of new draft revisions from approved source revisions.
- Updated simulated execution guard to require a current approved template
  revision for known template references.
- Updated Python local-agent client methods and tests.
- Added migration-reset coverage for the 0.8.x template table.
- Updated API, storage, local-agent, roadmap, README, changelog, release, and
  revision documentation.

## Modeling Notes

- Template identity is separate from content revision.
- Audit event order is not a business revision number.
- The definition checksum is the hash of canonical versioned JSON.
- Submitted and approved revisions are immutable; evolution starts from a new
  draft revision derived from an approved source.
- The 0.8.x table shape is intentionally reset. No compatibility runtime is
  maintained.

## Remaining Limits

- No campaign instantiation from templates.
- No acquisition, FFT, post-processing, or reporting runtime.
