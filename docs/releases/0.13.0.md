# Release 0.13.0 - Sensors, DAQ Channels, Scaling And Engineering Curves

Version `0.13.0` adds the reusable measurement-engineering layer between the
equipment catalog and a future acquisition runtime.

## Delivered

- Typed Rust core definitions for sensor/transducer definitions, scaling
  profiles, engineering curves, DAQ channel profiles, and acquisition channel
  recipes.
- Canonical JSON and SHA-256 checksums for all new definition revisions.
- Equipment SQLite migration `0003_sensors_scaling_curves.sql` with identity,
  revision, audit, and outbox evidence for the five new aggregate families.
- Local-agent routes for create, read, list, validate, save draft with checksum
  CAS, submit, approve, derive, clone, audit, and engineering-curve evaluation.
- Python client helpers and tests for the new public API.
- LAB CONSOLE editors for Sensors & Transducers, Scaling Profiles, Engineering
  Curves, DAQ Channels, and Acquisition Recipes.
- Engineering-curve CSV paste/import/export and deterministic 1D evaluation for
  frequency-dependent correction artifacts.
- Public-API seed scripts for current probe, biconical antenna, RF cable, RF
  amplifier, IEPE accelerometer, DAQ analog input, and `current_A` logical
  channel recipe.
- Playwright coverage for authoring and approving a complete logical
  measurement chain from scaling and curves through sensor, DAQ, and recipe.

## Domain Boundaries

- A sensor definition is a reusable sensor/transducer type, not a serialized
  physical asset.
- A scaling profile is a transformation contract, not a calibration event.
- An engineering curve is a machine-readable correction artifact, not the
  certificate or document store that justified it.
- A DAQ channel profile is a channel capability class, not a runtime channel.
- An acquisition channel recipe is a reusable logical channel setup, not a
  frozen campaign execution package.

## Examples Covered

- Current probe transfer and `10 mV/A` scaling.
- IEPE accelerometer scaling and excitation compatibility.
- Antenna factor versus frequency.
- RF cable loss and amplifier gain versus frequency.
- Analog input DAQ profile with sample-rate, range, coupling, and IEPE support.

## Validation Summary

The release validation target is the CI-equivalent script:

```powershell
.\scripts\validate-ci.ps1
```

Additional targeted checks used while building the release include Rust core
tests for the measurement-engineering domain, agent API tests, Python client
tests, seed idempotence against a local agent, LAB CONSOLE typecheck/lint/unit
tests, production build verification, and Playwright UI smoke/E2E checks.

## Known Limits

`0.13.0` does not implement real DAQ acquisition, oscilloscope acquisition,
spectrum acquisition, live station connection binding, physical asset fleet
redesign, campaign execution packages, FFT, report generation, RBAC,
authentication, central synchronization, or hardware runtime control.
