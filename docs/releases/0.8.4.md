# EMC Locus 0.8.4 - Test Template Lifecycle Transitions

Version `0.8.4` adds the first controlled lifecycle transitions for
agent-owned test templates.

This release builds directly on the 0.8.3 draft-template workflow. It does not
add another schema table and does not yet implement campaign instantiation,
configurable second approval, suspension, retirement, supersession, acquisition,
or post-processing execution.

## Delivered

- Added service support for `draft` -> `under_review`.
- Added service support for `under_review` -> `approved`.
- Added local API routes:
  - `POST /api/v1/test-templates/{template_id}/transitions/submit-for-review`;
  - `POST /api/v1/test-templates/{template_id}/transitions/approve`.
- Added structured refusal for unsupported lifecycle moves with
  `test_template_transition_not_allowed`.
- Added idempotent operation replay and conflict detection for lifecycle
  operations.
- Added audit actions:
  - `test_template_submitted_for_review`;
  - `test_template_approved`.
- Added pending outbox operations in domain `test_definitions` for both
  lifecycle actions.
- Added Python `LocalAgentClient` methods for submitting and approving test
  templates.
- Updated `docs/test-template-api.md` and local-agent documentation.

## Lifecycle Model

The supported lifecycle is intentionally small:

```text
draft -> under_review -> approved
```

The route names encode the business action instead of exposing a generic status
setter. Direct approval from `draft` is refused. Re-approval and unsupported
status jumps are also refused.

Every transition requires:

- `actor`;
- `reason`;
- `operation_id`;
- optional `correlation_id`;
- optional `device_id`.

The operation payload is kept stable for idempotent replay, while the audit row
records the action, actor, reason, base revision, resulting revision, and
payload checksum.

## Limits

- No configurable second-approval policy yet.
- No role or competence authorization check yet.
- No lifecycle route for suspension, retirement, or supersession.
- No campaign/test-instance creation from approved templates yet.
- No acquisition or post-processing execution.
