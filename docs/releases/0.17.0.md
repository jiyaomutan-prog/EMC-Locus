# Release 0.17.0 - Physical Measurement Setup Preparation

Version 0.17.0 connects the equipment and metrology work to the first useful
Locus Test Station workflow. An operator can now prepare and freeze the real
physical chain that will later support an EMC test.

## Delivered

- A typed Rust core aggregate for a revisioned physical measurement setup.
- Real asset bindings pinned to each asset revision and approved equipment
  model revision/checksum.
- Typed physical connections between equipment input, output and through
  ports.
- Explicit selection of a serial-specific time conversion or frequency
  response by characterization id and checksum.
- Contextual readiness for structure, identity, serviceability, calibration,
  evidence, nonconformity, port compatibility and correction applicability.
- A separate `station.sqlite` domain with deterministic revisions, canonical
  JSON, SHA-256 checksums, compare-and-swap draft replacement, immutable ready
  revisions, audit, operation replay and sync outbox evidence.
- Public local-agent routes and Python client methods for the complete
  workflow.
- A focused Qt `Préparation du poste` view for real materials, roles,
  connections, serial-specific corrections and the `Prêt à câbler` gate.
- Unit, route, real HTTP restart, migration and visual acceptance coverage.

## Signal Vocabulary

The operator chooses the correction from the data representation:

- **Conversion temporelle**: transforms each time sample using gain, offset or
  a controlled mapping, with optional overload/clipping limits.
- **Réponse fréquentielle**: compensates amplitude versus frequency and can
  also carry phase versus frequency.

Sensor `excitation` is presented as **Alimentation / conditionnement du
capteur**. The internal aggregate and route names remain technical contracts;
they are not used as the name of the operator's work.

## RF Cable Example

An approved cable model declares two 50-ohm RF through ports and its usable
band from `Fmin` to `Fmax`. The real cable dossier stores a metrologist-managed
table such as 10 MHz / 0.2 dB, 100 MHz / 1.0 dB and 1 GHz / 3.0 dB. The station
setup pins that exact characterization and verifies its validity on the
planned-use date.

## Evidence

The real HTTP scenario creates a draft, binds two real assets, connects the RF
cable to the receiver, selects the cable's measured loss response, obtains a
ready verdict, freezes the revision, checks audit/outbox, restarts the agent and
reloads the same immutable definition.

Qt captures at 1440 x 900 and 1280 x 720 verify that material identity, serial
number, connection ports, chosen response and readiness remain readable without
overlap.

## Limits

- The setup is not yet attached to a campaign test instance or execution
  package.
- No hardware is controlled and no samples or spectra are acquired.
- The selected correction is retained but is not yet applied to data.
- Readiness validates a selected serial-specific correction but does not yet
  infer from the equipment model that a particular path requires one.
- No FFT, result evaluation, report generation, authentication, RBAC,
  electronic signature or final network synchronization is added.
- Automated evidence demonstrates software behavior, not laboratory fitness
  or EN ISO/IEC 17025 conformity.
