# EMC Locus 0.6.2 - Rust Metrology DTO and Repository Boundary

## Delivered

Version `0.6.2` adds the first Rust boundary for the metrology vertical slice.
It does not yet expose the full agent-backed write workflow, but it makes Rust
capable of reading the 0.6.1 metrology schema through typed structures:

1. `metrology_dto.rs` defines Serde DTOs for instruments, lists, detail
   envelopes, and latest calibration records;
2. `metrology_repository.rs` opens and validates `metrology.sqlite`, checks the
   serviceability migration columns, loads instruments, and selects the latest
   calibration record;
3. `metrology_service.rs` renders deterministic JSON for instrument list/get
   use cases.

## Remaining Limits

- The new Rust metrology service is a read boundary only.
- Instrument registration, calibration recording, serviceability writes,
  idempotence, audit, outbox, HTTP routes, and Qt agent migration remain for the
  next 0.6.x tranches toward `0.7.0`.
