# EMC Locus 0.6.0 - Hardened Project Vertical Slice

## Delivered

Version `0.6.0` finalizes the first hardened local-agent project slice:

1. initialize local `projects.sqlite` and `sync.sqlite`;
2. enforce rollback-journal compatibility for project/sync atomic writes;
3. create a project through the Rust agent;
4. read project list/detail through the local agent;
5. read contract-review status through the local agent;
6. reject premature transition to `test_planning`;
7. complete required contract-review items;
8. transition to `test_planning`;
9. read project audit events through the local agent;
10. read pending sync outbox through the local agent;
11. replay an operation idempotently only when the canonical fingerprint matches;
12. reject a reused `operation_id` with a different canonical payload;
13. restart the agent and verify persisted state;
14. run Qt/Python project reads and writes through `agent_url` when configured;
15. validate the same matrix locally and in GitHub Actions CI.

## Remaining Limits

- Service planning is still a legacy SQLite-backed surface until an agent route
  is added.
- Metrology, test definitions, measurement data, update catalog, instrument
  control, acquisition, reporting, central synchronization, PostgreSQL, and
  object storage remain outside this hardened project slice.
- GitHub Actions CI is defined and pushed; remote run status depends on GitHub
  Actions availability for the repository.
