# EMC Locus 0.6.6 - Qt Metrology Agent Path

## Delivered

Version `0.6.6` migrates the temporary Qt/Python metrology surface onto the
local Rust agent when `agent_url` is configured:

1. the Python local-agent client now covers metrology reads, calibration reads,
   computed status, readiness, and audit-event routes;
2. Qt/Python instrument bootstrap rows come from the agent instead of
   `metrology.sqlite` in agent mode;
3. Qt shows a simple readiness table sourced from the Rust readiness service;
4. instrument registration, calibration-event recording, and serviceability
   changes use the agent without opening `metrology.sqlite` directly;
5. certificate references from the Qt/Python path can be forwarded as
   calibration-event document manifests;
6. legacy standalone document attachment remains disabled in agent-only mode
   until a dedicated document route exists.
