# Release 0.12.1 - CI Validation Parity And Release Pipeline Repair

Date: 2026-07-11

## Summary

`0.12.1` is a focused patch release for CI parity. It does not add product
features, drivers, acquisition, sensors, DAQ channels, scaling curves, station
bindings, a chain builder, RBAC, or sync behavior.

GitHub Actions for the pushed 0.12.0 release failed in workflow `CI`, job
`Local validation matrix`, step `Verify versioned LAB CONSOLE dist`. The
failing command was:

```powershell
git diff --exit-code -- apps/lab-console/dist
```

The root cause was Windows checkout line-ending conversion. The runner checked
out LAB CONSOLE HTML as CRLF, while Vite regenerated `dist/index.html` with LF
output during `npm run build`, so the committed production bundle was not
byte-stable under the exact CI path.

## Fixed

- Added `.gitattributes` rules for LAB CONSOLE HTML, CSS, JS, JSON, TS and TSX
  files so source and generated text artifacts remain LF on Windows checkout.
- Kept the `git diff --exit-code -- apps/lab-console/dist` check intact; the
  release now fixes deterministic output instead of weakening CI.
- Standardized active release validation on npm. `package-lock.json` remains
  the committed dependency graph and CI continues to run `npm ci`.
- Removed the LAB CONSOLE rebuild fallback to pnpm from `scripts/build-lab.ps1`.

## CI Diagnostics

- `.github/workflows/ci.yml` now prints `rustc`, `cargo`, `py`, `node`, and
  `npm` versions after tool setup.
- CI uploads failure diagnostics from `logs/launchers/**`,
  `apps/lab-console/test-results/**`, and
  `apps/lab-console/playwright-report/**` when validation fails.
- `scripts/validate-ci.ps1` mirrors the GitHub Actions command order for local
  pre-push validation and supports `-SkipE2E`, `-SkipSmoke`, and `-NoInstall`.
