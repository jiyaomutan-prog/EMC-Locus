# EMC Locus 0.9.2 - Windows Launcher Reliability

Version `0.9.2` fixes the Windows launchers introduced in `0.9.1` when the
repository path contains spaces, for example `E:\projets codex\EMC-Locus` or
`C:\EMC Locus Test Workspace\EMC-Locus`.

## Fixed

- `Start-Process -ArgumentList` no longer receives absolute repository paths
  containing spaces when relative paths are sufficient.
- `start-proto.ps1` waits for a real HTTP 200 before opening the browser or
  reporting success.
- `start-agent-qt.ps1` validates cargo, Python, storage initialization, agent
  health, and the storage root returned by `/api/v1/health`.
- Qt is never launched by `start-agent-qt.ps1` until the agent is healthy and
  bound to the requested local storage.
- Existing agents on the requested port are refused when they use another
  storage root.
- BAT wrappers keep the window open only after failures and propagate the
  PowerShell exit code.

## Added

- `scripts/launcher-common.ps1` shared launcher utilities.
- `scripts/stop-proto.ps1` and `.bat`.
- `scripts/stop-agent.ps1` and `.bat`.
- `scripts/stop-all.ps1` and `.bat`.
- `scripts/smoke-launchers.ps1` real launcher smoke tests.
- PID/state files under `logs\launchers\runtime`.

## Non-Goals

- No Template Studio.
- No new GUI framework.
- No new business domain.
- No acquisition, FFT, reporting, RBAC, or central synchronization.
