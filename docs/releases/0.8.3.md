# EMC Locus 0.8.3 - Agent-Owned Test Template Drafts

Version `0.8.3` adds the first executable backend slice for the template
backbone: controlled draft test templates written through the Rust local agent.

This release deliberately stops before method approval, template lifecycle
transitions, campaign instantiation, instrument runtime, acquisition, FFT,
reporting, or final sync. The goal is to move one LAB CONSOLE definition object
behind the same local-first write boundary used by projects, metrology,
documents, and simulated execution.

## Delivered

- Added `storage/sqlite/test_definitions/0003_test_templates.sql`.
- Added `test_templates` for draft executable test definitions.
- Added `test_template_audit_events`.
- Added Rust DTO, repository, and service modules for test templates.
- Added local API routes:
  - `POST /api/v1/test-templates`;
  - `GET /api/v1/test-templates`;
  - `GET /api/v1/test-templates/{template_id}`;
  - `GET /api/v1/test-templates/{template_id}/audit-events`.

- Added idempotent operation replay and conflict detection.

- Added `docs/test-template-api.md`.

## Template Model

Each draft test template records:

- stable `template_id`;
- `template_revision`;
- title and description;
- test category;
- measurement axis;
- optional method code and method revision;
- status, currently created as `draft`;
- variable definitions;
- variable lock policy;
- instrumentation chain slots;
- execution sequence;
- limits;
- post-processing definitions;
- creator and timestamps.

The structured template sections are stored as metadata JSON. They describe what
future TEST CONSOLE execution packages should contain; they are not execution
data.

## Limits

- New templates must start as `draft`.
- No approval or second-approval workflow.
- No lifecycle transition route.
- No project/campaign/test-instance instantiation.
- No method authoring route.
- No real acquisition or post-processing execution.
- No document attachment shortcut; controlled files remain in the
  attached-document registry.
