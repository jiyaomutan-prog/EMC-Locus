# Release 0.11.0 - Equipment Definition Catalog & Driver Script Studio

Date: 2026-07-03

## Summary

`0.11.0` adds the first real equipment engineering repository. Equipment model
definitions and driver profiles are now revisioned aggregates in Rust core,
persisted in a separate `equipment.sqlite` repository, exposed by local-agent
routes, available through the Python client, and editable from LAB CONSOLE.

This release is deliberately not a certified hardware runtime. VISA, vendor
CAN, USBTMC and HID contracts are modeled and simulated, but unavailable real
providers are reported as unavailable.

## Added

- `equipment.sqlite` domain with model identities/revisions, driver
  identities/revisions, audit events, registries, indexes and active-draft
  guards.
- Typed equipment/driver definitions in `emc-locus-core`: specifications,
  physical ports, interfaces, capabilities, actions, safety classes, structured
  scripts, CAN frames, canonical JSON, checksums and validation.
- Deterministic driver simulation using the same AST shape as the future
  runtime.
- Local-agent APIs for equipment models, driver profiles, validation,
  transitions, audit, provider status and simulation.
- Python `LocalAgentClient` methods and tests for the new API.
- LAB CONSOLE Equipment space with Model Catalog and Drivers and Actions.
- Idempotent seed scripts for SCPI power meter, serial amplifier, CAN power
  unit and manual antenna examples.

## Changed

- `storage init/status/verify` now includes `equipment.sqlite`.
- Template Studio labels now distinguish business library title from technical
  revision title.
- Template clone now updates the cloned technical definition title to match the
  new business title by default.
- LAB launchers can seed equipment demo data without requiring proprietary
  VISA/CAN/USB providers.

## Validation

The final release validation is recorded in the session log and revision
control documentation. Unit and integration tests validate the model, API,
Python client, migrations, frontend and launcher smoke paths. These tests do
not certify operation with real instruments.

## Remaining Limits

- No physical fleet redesign or station connection binding.
- No certified VISA, vendor CAN, USB, DAQ or acquisition runtime.
- No real-time acquisition, FFT, report generation, authentication or RBAC.
- Driver scripts are structured and simulated, but not executed against live
  equipment in this release.
