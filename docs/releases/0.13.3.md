# Release 0.13.3 - LAB CONSOLE Experience Foundation

Version `0.13.3` is a focused application-shell and operator-experience release.
It does not add a new laboratory domain, equipment runtime, or acquisition
capability.

## Delivered

- Replaced the fourteen-destination sidebar with focused active navigation for
  Methods, Equipment, and local System status.
- Kept future Metrology, Planning, Campaigns, and Reports verticals visible as
  non-actionable roadmap context.
- Added a collapsible desktop sidebar, compact sticky top bar, accessible icon
  controls, visible focus states, and responsive tablet/mobile navigation.
- Made Equipment controls contextual and removed duplicated measurement-
  engineering tab rows.
- Converted the equipment model wizard into a focused modal workflow.
- Prioritized model names, human revision states, revision numbers, and current
  lifecycle actions over IDs and checksums in normal views.
- Moved traceability context and technical catalog filters behind explicit
  progressive disclosures.
- Preserved usable editor width and stopped sparse cards and empty states from
  stretching across the viewport.
- Included the post-`0.13.2` service-schedule overlap guards for normalized
  imported time windows.

## Validation Coverage

- Frontend unit tests cover active/future navigation, sidebar collapse and
  reopen, contextual catalog controls, and existing editor behavior.
- Playwright covers method creation through approval and derivation, Equipment
  taxonomy/model creation, the modal wizard, and a complete approved
  measurement-engineering recipe workflow.
- The release follows the repository CI-equivalent script for Rust, Python,
  SQLite, frontend, launcher, release-consistency, and whitespace checks.

## Explicit Limits

- No physical serial-numbered asset fleet or station wiring.
- No real acquisition, driver execution, FFT, reporting, RBAC, or central sync.
- No full method-editor redesign; advanced editors still expose some technical
  vocabulary.
- No formal WCAG, usability-panel, or EN ISO/IEC 17025 compliance validation.
