# Release 0.12.0 - Equipment Physics Classification Productization

Date: 2026-07-11

## Summary

`0.12.0` productizes the physics-based equipment classification work started on
the 0.11 equipment catalog. Classification is now backend-owned, searchable,
indexed, documented, and usable from LAB CONSOLE model creation workflows.

This release does not add a new hardware runtime. It does not implement sensor
scaling, DAQ acquisition, engineering curves, a physical asset fleet redesign,
station connection binding, a measurement-chain builder, reports, RBAC, or
central synchronization.

## Added

- SQLite equipment migration `0002_physical_classification.sql` with
  functional-role, signal-domain, port-directionality, flow-role and
  technology-tag registries.
- Backend classification presets and preset port templates for energy sources,
  signal sources, RF equipment, sensors, actuators, measurement instruments,
  converters/acquisition, processing/control systems and software systems.
- Indexed equipment model classification summaries plus normalized
  signal-domain and technology-tag summary tables maintained transactionally by
  the agent.
- Local API routes for classification registries, preset listing, preset detail,
  and draft model creation from preset.
- Catalog filters for functional role, signal domain, technology tag,
  equipment class, manufacturer, status and free text.
- Python client methods for registries, presets, preset creation and filtered
  model listing.
- LAB CONSOLE Model Catalog filters, creation panel, preset preview,
  Classification section, and structured Port Topology editing.
- Core validation hardening for ADC/DAC/CAN bus ambiguity, CAN bus protocol
  topology, RF impedance, through devices, sensors, measurement instruments,
  acquisition devices and software-only models.

## Changed

- Version metadata is synchronized to `0.12.0` across Rust, Python and LAB
  CONSOLE.
- Demo equipment seeding now uses the public API and creates representative
  classified models from backend presets.
- Documentation now separates equipment class, functional role, signal domain,
  access provider, technology tag, port directionality and flow role.

## Remaining Limits

- No real VISA, CAN bus, USB, serial or DAQ hardware session is opened by this
  release.
- Presets create editable draft model definitions only; they do not create
  approved models, driver profiles or station bindings.
- Classification summaries are indexed catalog views of the latest relevant
  model revision, not an execution package snapshot.
- Sensor definitions, DAQ channels, scaling, antenna factors, cable loss
  curves, transfer functions and uncertainty curves remain future work.

## Recommended Next Vertical

`0.13.0 - Sensors, DAQ Channels, Scaling And Engineering Curves`

That vertical should cover sensor/transducer definitions, DAQ channels,
two-point and function-based scaling, lookup tables, transfer curves,
antenna-factor curves, cable-loss curves, gain/uncertainty curves,
frequency-dependent correction artifacts, and interactive calibration capture.
