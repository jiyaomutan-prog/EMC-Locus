# EMC Locus 0.9.0 - Revisioned Test Template Aggregate

Version `0.9.0` replaces the provisional 0.8.x test-template model with a
revisioned business aggregate.

This release is intentionally a modeling and workflow release. It does not add
a LAB CONSOLE template editor, campaign instantiation, real acquisition,
post-processing execution, FFT, reporting, authentication, RBAC, or a new test
runtime.

## Delivered

- Added a typed test-template definition module in `emc-locus-core`.
- Added structural and semantic validation for variables, defaults, lock
  policies, instrumentation slots, sequence branches, limits, and
  post-processing dependencies.
- Added deterministic canonical JSON and `sha256:` definition checksums.
- Added SQLite migration `0004_template_revision_aggregate.sql` with:
  - `test_template_identities`;
  - `test_template_revisions`;
  - `test_template_audit_events`.
- Replaced the local-agent template service, repository, DTOs, and routes with
  the revisioned workflow.
- Added draft definition replacement with `expected_definition_checksum`.
- Made `under_review` and `approved` revisions immutable.
- Added derivation of a new draft from an approved source revision.
- Added audit and outbox records with explicit revision ids and old/new
  definition checksums.
- Updated the Python local-agent client for the new API.
- Added real HTTP E2E coverage for create, edit, submit, approve, derive,
  audit/outbox, restart, and re-read.

## Migration Policy

The software has not been commercialized. Version 0.9.0 deliberately resets the
0.8.3/0.8.4 `test_templates` storage shape. The migration drops the provisional
template tables and creates the new identity/revision/audit model.

There is no dual-read, dual-write, or legacy DTO in the 0.9.0 runtime.

## Approval Scope

The API continues to require `actor`, `reason`, and `operation_id`.

It does not yet implement authenticated identity, permissions, competence
checks, separation of author and approver, or configurable second approval.
Those rules belong to the future people/roles/competence domain.

## Remaining Limits

- No graphical LAB CONSOLE template editor.
- No campaign test instance created from a template.
- No execution package snapshot that resolves variables into a run.
- No real instrument driver or acquisition path in this release.
- No post-processing engine or report output.
- No central synchronization merge policy.
