# EMC Locus 0.8.2 - GUI And Template Backbone

Version `0.8.2` freezes the GUI and template backbone before adding another
major runtime vertical.

This release does not add Rust routes, SQLite schemas, instrument drivers,
acquisition, FFT, reporting, object upload, or final synchronization. It
clarifies the product model that future LAB CONSOLE and TEST CONSOLE work must
follow.

## Delivered

- Accepted ADR 0002 as the active decision for the LAB/TEST console split and
  template backbone freeze.
- Clarified that LAB CONSOLE is the web-oriented laboratory management console.
- Clarified that TEST CONSOLE is the Qt local/offline execution console.
- Reframed metrology as a controlled LAB domain and a TEST readiness dependency,
  not a third GUI product.
- Strengthened:
  - `docs/gui/gui-dual-surface-architecture.md`;
  - `docs/gui/lab-console-information-architecture.md`;
  - `docs/gui/test-console-workspace.md`;
  - `docs/domain/template-and-execution-definition.md`;
  - `docs/domain/people-roles-competence.md`;
  - `docs/adr/0002-gui-and-template-backbone.md`.
- Updated ADR 0003 so its document-registry decision remains valid without
  keeping the superseded three-GUI-product wording.
- Refactored `apps/gui-shell` as a static LAB CONSOLE information-architecture
  prototype:
  - hierarchical LAB navigation;
  - LAB/TEST boundary cards;
  - backbone and guardrail panels;
  - object-space views with objectives, handoff notes, guardrails, relations,
    actions, and structural examples.

## Product Boundary

LAB CONSOLE owns laboratory management: clients, products, projects, campaigns,
templates, methods, documents, people, roles, competences, metrology, planning,
reports, sync, audit, and updates.

TEST CONSOLE owns execution: local/offline station context, readiness,
instrumentation chain, acquisition, monitoring, deviations, substitutions,
reruns, and execution evidence publication.

The static shell must not receive fake runtime behavior. It can demonstrate
information architecture and object relationships only.

## Limits

- No backend refactor.
- No new Rust runtime slice.
- No SQLite migration.
- No real acquisition, drivers, FFT, Parquet, HDF5, or reporting.
- No final web framework selection.
- No legacy compatibility expansion.
