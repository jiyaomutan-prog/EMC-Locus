# Release 0.13.2 - Equipment Repository Usability Polish

Version `0.13.2` is a focused LAB CONSOLE usability patch for the Equipment
Repository. It does not start the `0.14.0` physical asset, station wiring, or
measurement-chain vertical.

## Delivered

- Grouped the Equipment workspace navigation into Referentiel, Ingenierie de
  mesure, Pilotage, and A venir.
- Replaced action-looking category choices in the new model wizard with
  explicit radio/list choices and a tree-based subcategory step.
- Rebuilt category rendering as a real tree with indentation, folder icons,
  expand/collapse, selected row state, hover state, keyboard selection, and
  `...` contextual actions.
- Reworked Repository Administration around the selected category:
  Informations, Sous-categories, Formulaire, Previsualisation, and Diagnostic
  avance.
- Added label-first category and field creation with generated internal codes
  hidden behind advanced options.
- Added a visible choice-list editor for field option values.
- Hid checksums, field codes, category IDs, revision IDs, schema versions, and
  raw JSON from the normal creation and preview flows.
- Extended catalog filtering so a category filter includes arbitrary-depth
  descendant categories.

## Validation Coverage Added

- Frontend unit coverage for non-button category choices, nested category
  administration, contextual category actions, generated field codes, hidden
  diagnostics, choice-list editing, form preview, and wizard creation.
- Playwright coverage for seven root categories in the tree, nested RF category
  creation, field creation without manual `field_code`, form preview, model
  creation in a nested category, and category subtree filtering.
- Rust local-agent regression coverage for nested categories and subtree model
  filtering.

## Explicit Non-Goals

- No physical serial-numbered asset fleet.
- No station connection binding.
- No graphical measurement-chain builder.
- No real acquisition, FFT, hardware runtime, reporting, authentication/RBAC,
  or central synchronization.
