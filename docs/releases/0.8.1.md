# EMC Locus 0.8.1 - Agent-Owned Attached Documents

Version `0.8.1` adds the first shared attached-document registry behind the
Locus Local Agent.

This is not an upload system. The slice stores document metadata, ownership,
storage reference, checksum, revision, applicability, confidentiality, audit,
and outbox evidence. File bytes remain outside SQLite, aligned with the target
architecture where PostgreSQL/SQLite keep metadata and object storage keeps
documents and scientific payloads.

## Document Model

Each document records:

- stable `document_id`;
- `classification`;
- `title`;
- owner surface/domain;
- owner entity type and id;
- storage backend and URI;
- original filename and MIME type;
- byte size;
- SHA-256 checksum;
- revision;
- applicability;
- confidentiality;
- creator and timestamps.

When the owner is a Locus Lab Management project, the project must already
exist. This keeps project document attachments traceable without pretending the
current slice owns every future document domain.

## Limits

- No file upload or download.
- No object-storage service integration.
- No PDF/Word/Excel/image parsing.
- No Parquet or HDF5 dataset writing.
- No central PostgreSQL.
- No final document permission model.
- The current outbox domain is still `project_records` with entity type
  `attached_document`; a future slice may promote documents to a dedicated
  repository domain.
